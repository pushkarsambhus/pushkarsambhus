# Coding Interview Prep — Python

A structured list of problems to practice for cracking coding interviews.
No solutions included — work through each problem yourself!

## Structure

| File | Difficulty | Problems |
|------|------------|----------|
| `easy_problems.py` | Easy | 35 problems |
| `medium_problems.py` | Medium | 45 problems |
| `hard_problems.py` | Hard | 40 problems |

## Topics Covered

### Easy
- Arrays & Hashing
- Strings
- Stack & Queue
- Binary Search
- Linked List
- Binary Trees
- Dynamic Programming
- Math & Bit Manipulation

### Medium
- Two Pointers & Sliding Window
- Backtracking
- Graphs (BFS/DFS)
- Binary Search (advanced)
- Trees (BST, LCA, construction)
- Dynamic Programming
- Intervals & Greedy
- Heaps

### Hard
- Monotonic Stack
- Trie
- Advanced DP (Edit Distance, Regex, Burst Balloons)
- Graph Algorithms (Dijkstra, Topological Sort, Bridges)
- Two Heaps
- Advanced Backtracking (N-Queens, Sudoku Solver)
- LRU/LFU Cache Design

## Suggested Study Order

1. Start with all **Easy** problems to build fundamentals
2. Move to **Medium** — these are the most common in interviews (FAANG, startups)
3. Tackle **Hard** only after you're comfortable with Medium

## Tips

- Always think about **time and space complexity** before coding
- Try to solve each problem in multiple ways (brute force → optimized)
- Practice explaining your approach out loud
- Re-attempt problems you struggled with after a few days
